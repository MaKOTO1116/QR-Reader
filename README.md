# 入退室QRリーダーWebアプリ (for iPad Safari)

塾やスクール等の入退室処理をiPadのSafariブラウザ上で行うための、静的Webアプリケーションです。

---

## 1. アプリの目的と動作フロー

### 目的
iPadのインカメラ（内側カメラ）で生徒のQRコードをスキャンし、QRコード内に埋め込まれた本部システムのURLへアクセスすることで、チェックイン（入室・退室）処理を行います。

### 動作フロー
1. **QRコードの読み取り**: カメラ映像からQRコードをスキャンします。
2. **URLの抽出と検証**: QRコードからURLを取得し、不正なスキームや許可されていないドメインがないか検証します。
3. **隠し iframe によるアクセス**: 検証を通過したURLを、画面上に表示されない非表示の `<iframe>` に設定してアクセスします。
4. **処理完了検知・完了音**: iframeの読み込み完了（`load` イベント）を検知すると、電子音（成功チャイム）を再生します。
5. **スキャナー自動再開**: 同一QRコードの二重読み取りを防ぐためのクールダウン（5秒間）および成功通知画面の表示後、自動的に次のスキャン状態へと戻ります。

---

## 2. 実行環境と動作前提

* **iPad / iPadOS Safari**での使用を第一に最適化されています。
* **HTTPS接続が必須**です（ローカルテスト時の `localhost` / `127.0.0.1` を除く）。ブラウザのセキュリティ制限（WebRTC/getUserMedia）により、暗号化されていないHTTP接続ではカメラが起動しません。
* 外部サーバーサイド処理を持たない**静的なWebアプリ**（HTML, CSS, JavaScriptのみ）として動作するため、GitHub Pages等で簡単にホスト可能です。
* **本部システムへのログインが前提**: 本アプリを使用するiPad Safariで、事前に本部システム（QRコードの遷移先システム）へログインを済ませておいてください。

---

## 3. 重要事項と制限事項（ご利用前に必ずお読みください）

本アプリは静的HTMLの `<iframe>` 読み込み機能を利用して動作するため、ブラウザのセキュリティ機能（Same-Origin Policy等）に伴う以下の制約があります。

1. **URL読み込み完了 ＝ 処理成功の不保証**
   * 本アプリは iframe の読込完了 (`load` イベント) を検知して完了音を鳴らします。これは「サーバーから応答があったこと」を意味しますが、**「入退室処理自体がシステム的に成功したこと」は保証しません**。例えば、本部システム側でセッション切れエラー画面が表示されたり、サーバーエラー（500）が発生した場合でも、ブラウザは「読み込み完了」とみなすため、成功音が鳴ります。
2. **クロスオリジンによる情報不透過性**
   * QRコードのURL（本部システム）と、本アプリがホストされているドメインが異なる場合（クロスオリジン）、ブラウザの同一生成元ポリシー（Same-Origin Policy）により、アプリ側から iframe の内部構造（表示内容、ログイン成否、エラー文言等）を読み取ることは一切できません。
3. **X-Frame-Options による動作拒否**
   * 本部システム側がセキュリティヘッダー `X-Frame-Options: DENY` または `SAMEORIGIN` を設定している場合、Safariは iframe 内でのページ表示を完全にブロックします。この制限がかかっている場合、本アプリ（ブラウザ版）は動作しません。
4. **SafariのCookie制限 (ITP)**
   * Safariのインテリジェント・トラッキング防止機能（ITP）およびサードパーティCookieの制限により、iframe内で本部システムのログイン状態が維持されない場合があります。
5. **専用アプリへの移行推奨**
   * 上記 3〜4 の制限により動作しない、あるいは厳密な処理成否判定を行いたい場合は、Safariブラウザではなく、`WKWebView` を組み込んだiOS/iPadOS専用のネイティブアプリを構築し、URLアクセス時のナビゲーションヘッダーの変更やCookie制御を個別に行う必要があります。

## 4. iPadでの使用方法

### カメラ権限の許可
1. 初めてアプリを開いた際、または「カメラ開始」ボタンを押した際に「カメラへのアクセスを求めています」とダイアログが表示されるので、**「許可」**を選択してください。
2. もし拒否してしまった場合は、iPadの `「設定」アプリ ＞ 「Safari」 ＞ 「カメラ」` に進み、アクセス権を **「確認」** または **「許可」** に変更してください。

### ホーム画面への追加（PWA化）
iPadのホーム画面にアプリアイコンを登録することで、全画面表示で使用でき、操作性・視認性が向上します。
1. Safariで本アプリのURLを開きます。
2. 画面右上（または下部）の **「共有（アクション）」ボタン**（四角から上矢印が出ているアイコン）をタップします。
3. メニューから **「ホーム画面に追加」** を選択します。
4. ホーム画面にアイコンが作成され、次回以降はそこから起動できます。
   * *注意: iPadOSのバージョンによっては、ホーム画面アプリモードにおいてカメラパーミッション確認が起動するたびに発生する場合があります。その際はSafariブラウザから直接ご使用ください。*

### 音声（完了音）が鳴らない場合の確認事項
本アプリは Web Audio API を用いてブラウザ内部でシンセサイザー音を合成し鳴らしています。音が鳴らない場合は以下を確認してください。
* iPadの側面スイッチまたはコントロールセンターで **消音モード（マナーモード）** になっていないか確認してください。
* iPadのシステム音量がミュートまたは極端に小さくなっていないか確認してください。
* 初回起動時に**「カメラ開始」**ボタンをタップすることで、ブラウザの音声再生セキュリティ制限（自動再生の保護）が解除されます。必ずユーザー操作を伴って開始してください。
* iPadの仕様により、画面が自動ロック（スリープ）されるとカメラおよび音声処理が中断されます。運用時は、iPadの `「設定」 ＞ 「画面表示と明るさ」 ＞ 「自動ロック」` を **「なし」** に設定することをお勧めします。

## 5. 設定値の変更方法

`app.js` の先頭にある `CONFIG` オブジェクトを変更することで、動作パラメーターを調整できます。

```javascript
const CONFIG = {
  TIMEOUT_DURATION: 15000,       // iframe読み込みタイムアウト時間（ミリ秒）
  COOLDOWN_DURATION: 5000,       // 同一URLの再受付禁止時間（ミリ秒）
  ALLOWED_PROTOCOLS: ['https:'], // 許可するプロトコル（初期状態はhttps:のみ）
  ALLOWED_HOSTS: ['user.edmondo.jp'], // 許可ドメイン（ドメイン制限を行う場合に指定）
  
  // 状態表示自動復帰のウェイト時間
  SUCCESS_RESET_DELAY: 3000,
  ERROR_RESET_DELAY: 4000,
  DUPLICATE_RESET_DELAY: 3000,
  
  DEBUG: false,                  // デバッグログ出力フラグ
  AUDIO_VOLUME: 0.6              // 完了音・警告音の音量 (0.0 〜 1.0)
};
```

### ドメイン制限の設定例
第三者の不適切なURLが読み取られた際、iframeで読み込まないようにセキュリティ保護を行うことができます。

**例：`user.edmondo.jp` のみ接続を許可する場合**
```javascript
ALLOWED_HOSTS: ['user.edmondo.jp'],
```
*空配列 `[]` の場合は、`ALLOWED_PROTOCOLS`（初期状態では `https:`）に合致するすべてのドメインへのアクセスを許可します。*

## 6. ライブラリについて

本アプリは、QRコードの画像解析処理のために以下の外部ライブラリを使用しています。

* **ライブラリ名**: `jsQR`
* **バージョン**: `1.4.0`
* **採用の理由**: iOS/iPadOS Safariの `getUserMedia` 映像ストリーム上で、非常に軽量かつ高い検知精度で動作実績があるため。
* **ローカル配置の実施**: CDN障害やネットワーク切断時のシステム停止リスクを排除するため、`jsQR.js` のソースコードはリポジトリ内にローカルファイルとして同梱されています（元データは `https://cdn.jsdelivr.net/npm/jsqr@1.4.0/dist/jsQR.min.js` からダウンロードしたものです）。

## 7. GitHub Pages への公開手順

本アプリは静的ファイルのみで構成されているため、GitHub Pages で瞬時に全世界（HTTPS）へ公開可能です。

1. 作成したコードを GitHub のリポジトリにプッシュします（ブランチ名は `main` 等）。
2. GitHub 上のリポジトリページを開き、**「Settings」** タブをタップします。
3. 左メニューの **「Pages」** を選択します。
4. **「Build and deployment」** セクションの **Source** で **「Deploy from a branch」** を選択します。
5. **Branch** に `main` (および `/ (root)`) を指定し、**「Save」** をクリックします。
6. 数分後、`https://<ユーザー名>.github.io/<リポジトリ名>/` の形式で公開アドレスが生成されます。
